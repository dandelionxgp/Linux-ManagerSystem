# 商品库存管理系统 — 设计开发计划

---

## 一、项目概述

### 1.1 项目名称
**商品库存管理系统**（超市库存管理系统）

### 1.2 项目目标
构建一套基于 Web 的商品库存管理平台，实现商品的入库、出库、查询、盘点、打印等核心业务功能，取代手工记账方式，提升库存管理效率与准确性。

### 1.3 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 操作系统 | Ubuntu Server LTS | 24.04 |
| 前端框架 | Vue 3 (Composition API) | 3.5+ |
| 构建工具 | Vite | 5.x |
| UI 组件库 | Element Plus | 2.x |
| HTTP 客户端 | Axios | 1.x |
| 后端语言 | PHP | 8.4 |
| 后端框架 | Laravel | 11.x |
| 数据库 | MySQL | 8.4 |
| Web 服务器 | Nginx | 1.26+ |

### 1.4 开发环境搭建（完整指南）

由于开发目标平台是 Ubuntu 24.04 LTS，而你的本地环境是 Windows 11，推荐使用 **WSL2** 方案，无需额外安装 Ubuntu 物理机或虚拟机，即可在 Windows 上获得原生 Linux 开发体验。

---

#### 1.4.1 方案选择

| 方案 | 说明 | 推荐度 |
|------|------|--------|
| **WSL2 + Ubuntu 24.04** ⭐ | Windows 内嵌 Linux，VS Code 原生集成，性能等同于原生 | ★★★★★ |
| **Docker Desktop** | 容器化各服务，环境可复现，适合团队协作 | ★★★★ |
| **VirtualBox 虚拟机** | 完全隔离环境，资源占用较大 | ★★★ |
| **远程 Linux 服务器** | SSH 远程开发，需额外服务器 | ★★★ |

**本计划采用 WSL2 + Ubuntu 24.04 作为主方案，Docker 作为备选方案。**

#### 1.4.2 方案一：WSL2 + Ubuntu 24.04（主方案）

##### Step 1 — 启用 WSL2

在 Windows **PowerShell（管理员身份）** 中依次执行：

```powershell
# 1. 启用 WSL 功能（Windows 10 2004+ / Windows 11 自带）
wsl --install --no-distribution

# 2. 设置 WSL2 为默认版本
wsl --set-default-version 2

# 3. 安装 Ubuntu 24.04 LTS
wsl --install -d Ubuntu-24.04

# 4. 重启电脑
```

重启后，从开始菜单打开 **Ubuntu 24.04**，首次启动会提示创建 Linux 用户名和密码（自行设定，牢记）。

##### Step 2 — Ubuntu 系统初始化

```bash
# 更新包管理器
sudo apt update && sudo apt upgrade -y

# 设置中文时区（可选）
sudo timedatectl set-timezone Asia/Shanghai

# 安装常用工具
sudo apt install -y curl wget git unzip build-essential software-properties-common
```

##### Step 3 — 安装 PHP 8.4

```bash
# 添加 PHP PPA 源
sudo add-apt-repository ppa:ondrej/php -y
sudo apt update

# 安装 PHP 8.4 及必要扩展
sudo apt install -y \
  php8.4 \
  php8.4-fpm \
  php8.4-cli \
  php8.4-mbstring \
  php8.4-xml \
  php8.4-curl \
  php8.4-mysql \
  php8.4-bcmath \
  php8.4-zip \
  php8.4-intl \
  php8.4-gd

# 验证安装
php -v        # 应显示 PHP 8.4.x
php -m        # 查看已安装扩展列表
```

##### Step 4 — 安装 Composer（PHP 包管理器）

```bash
# 下载并全局安装 Composer
curl -sS https://getcomposer.org/installer | php
sudo mv composer.phar /usr/local/bin/composer

# 验证
composer --version

# 配置国内镜像（加速包下载，可选）
composer config -g repo.packagist composer https://mirrors.aliyun.com/composer/
```

##### Step 5 — 安装 MySQL 8.4

```bash
# 安装 MySQL Server
sudo apt install -y mysql-server-8.4

# 启动并设置开机自启
sudo systemctl start mysql
sudo systemctl enable mysql

# 检查运行状态
sudo systemctl status mysql

# 安全初始化（设置 root 密码、移除匿名用户等）
sudo mysql_secure_installation
# 按提示操作：设置 root 密码 → 输入新密码 → 全部选 Y

# 登录验证
mysql -u root -p
```

创建项目数据库和用户：

```sql
-- 在 MySQL 命令行中执行
CREATE DATABASE stock_manager CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'stockadmin'@'localhost' IDENTIFIED BY 'your_password_here';
GRANT ALL PRIVILEGES ON stock_manager.* TO 'stockadmin'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

##### Step 6 — 安装 Nginx

```bash
sudo apt install -y nginx

# 启动并设置开机自启
sudo systemctl start nginx
sudo systemctl enable nginx

# 验证（curl 返回 Welcome to nginx 页面）
curl http://localhost
```

##### Step 7 — 安装 Node.js 22.x（前端构建）

```bash
# 添加 NodeSource 仓库
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -

# 安装 Node.js（npm 随附）
sudo apt install -y nodejs

# 验证
node -v     # 应显示 v22.x.x
npm -v      # 应显示 10.x.x

# 配置 npm 国内镜像（加速包下载，可选）
npm config set registry https://registry.npmmirror.com
```

##### Step 8 — 安装 Git 并配置

```bash
sudo apt install -y git

# 配置用户信息
git config --global user.name "你的姓名"
git config --global user.email "你的邮箱"
```

##### Step 9 — VS Code 远程开发配置

1. 在 **Windows** 上安装 [VS Code](https://code.visualstudio.com/)
2. 打开 VS Code，在扩展市场搜索安装：
   - **WSL**（ms-vscode-remote.vscode-remote-extensionpack）
   - **Vue - Official**（Vue 3 语法支持）
   - **PHP Intelephense**（PHP 智能提示）
   - **Laravel Extra Intellisense**（Laravel 开发辅助）
3. 在 Ubuntu 终端中，进入项目目录后运行：
   ```bash
   code .
   ```
   VS Code 会自动以 **Remote-WSL** 模式连接，左下角显示 `WSL: Ubuntu-24.04` 即表示成功。

> **开发工作流**：在 Windows 的 VS Code 界面中编辑代码 → 终端自动连接到 Ubuntu → 所有命令（php/composer/npm/mysql）都在 Ubuntu 内运行。

##### Step 10 — Windows 侧工具安装

| 工具 | 用途 | 下载地址 |
|------|------|----------|
| VS Code | 代码编辑器 | code.visualstudio.com |
| DBeaver | 数据库可视化管理 | dbeaver.io |
| Postman / Bruno | API 调试 | postman.com / usebruno.com |
| Another Redis Desktop Manager（可选） | Redis 管理 | github.com/qishibo/AnotherRedisDesktopManager |

##### Step 11 — 环境验证清单

全部安装完成后，在 Ubuntu 终端中逐条验证：

```bash
# 验证清单（每一项都应正常输出版本号）
php -v                  # PHP 8.4.x
composer --version      # Composer 2.x
mysql --version         # MySQL 8.4.x
nginx -v                # nginx 1.26.x
node -v                 # v22.x.x
npm -v                  # 10.x.x
git --version           # git 2.x
curl http://localhost   # Nginx 欢迎页
php -m | grep pdo_mysql # 应输出 pdo_mysql
```

#### 1.4.3 方案二：Docker 统一环境（备选方案）

如果你不希望手动安装各组件，可以使用 Docker 一键启动整个开发环境。

##### 前提条件

在 Windows 上安装 [Docker Desktop](https://www.docker.com/products/docker-desktop/)，启用 WSL2 后端。

##### 项目 docker-compose.yml

在项目根目录创建 `docker-compose.yml`：

```yaml
version: '3.8'

services:
  # ========== MySQL 8.4 ==========
  mysql:
    image: mysql:8.4
    container_name: stock_mysql
    restart: unless-stopped
    environment:
      MYSQL_ROOT_PASSWORD: root123
      MYSQL_DATABASE: stock_manager
      MYSQL_USER: stockadmin
      MYSQL_PASSWORD: stock123
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
      - ./database/init.sql:/docker-entrypoint-initdb.d/init.sql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5

  # ========== PHP 8.4 + Laravel ==========
  php:
    build:
      context: ./docker/php
      dockerfile: Dockerfile
    container_name: stock_php
    restart: unless-stopped
    working_dir: /var/www/html
    volumes:
      - ./backend:/var/www/html
    depends_on:
      mysql:
        condition: service_healthy
    environment:
      DB_HOST: mysql
      DB_DATABASE: stock_manager
      DB_USERNAME: stockadmin
      DB_PASSWORD: stock123

  # ========== Nginx ==========
  nginx:
    image: nginx:1.26-alpine
    container_name: stock_nginx
    restart: unless-stopped
    ports:
      - "80:80"
    volumes:
      - ./backend/public:/var/www/html/public
      - ./docker/nginx/default.conf:/etc/nginx/conf.d/default.conf
    depends_on:
      - php

  # ========== Node.js (Vue 3 前端开发服务器) ==========
  node:
    image: node:22-alpine
    container_name: stock_node
    working_dir: /app
    volumes:
      - ./frontend:/app
    ports:
      - "5173:5173"
    command: sh -c "npm install && npm run dev -- --host 0.0.0.0"

volumes:
  mysql_data:
```

##### PHP Dockerfile（`docker/php/Dockerfile`）

```dockerfile
FROM php:8.4-fpm

RUN apt update && apt install -y \
    libpng-dev libonig-dev libxml2-dev zip unzip curl git \
    && docker-php-ext-install pdo_mysql mbstring exif pcntl bcmath gd

COPY --from=composer:latest /usr/bin/composer /usr/bin/composer

WORKDIR /var/www/html
```

##### Nginx 配置（`docker/nginx/default.conf`）

```nginx
server {
    listen 80;
    server_name localhost;
    root /var/www/html/public;

    index index.php index.html;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass php:9000;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }
}
```

##### 启动命令

```bash
# 在项目根目录执行
docker compose up -d

# 查看运行状态
docker compose ps

# Laravel 初始化（进入 PHP 容器执行）
docker exec -it stock_php bash
composer install
php artisan migrate
php artisan db:seed

# 前端访问地址
# 后端 API: http://localhost/api
# 前端页面: http://localhost:5173
```

#### 1.4.4 方案对比总结

| 对比维度 | WSL2 方案 | Docker 方案 |
|----------|-----------|-------------|
| 安装复杂度 | 中等（需手动安装各组件） | 低（一条命令启动） |
| 启动速度 | 快（服务常驻后台） | 中等（需拉起容器） |
| 学习成本 | 需了解各组件配置 | 需了解 Docker 基础 |
| 调试便利性 | 直接操作进程，方便断点调试 | 需进入容器操作 |
| 环境一致性 | 依赖本地配置 | **极好**，团队统一 |
| 适合场景 | 个人学习开发 | 团队协作 / 快速演示 |

> **建议**：如果这是你第一次接触 Linux 开发，用 **WSL2 方案**，边学边搭建，有助于深入理解各组件的工作原理。如果时间紧张或需要和他人协作，用 **Docker 方案**。

## 二、需求分析

### 2.1 功能模块总览

```
商品库存管理系统
├── 1. 商品清单管理
├── 2. 入库管理
├── 3. 出库管理
├── 4. 库存查询
├── 5. 库存盘点
├── 6. 打印表单
└── 7. 系统管理（用户/权限/日志）
```

### 2.2 功能详细说明

#### 模块一：商品清单管理
| 编号 | 功能 | 描述 |
|------|------|------|
| F-01 | 商品列表 | 分页展示所有商品，支持排序 |
| F-02 | 新增商品 | 录入商品编码、名称、分类、规格、单位、单价、安全库存量等 |
| F-03 | 编辑商品 | 修改已有商品信息 |
| F-04 | 删除商品 | 逻辑删除（有出入库记录的商品不可物理删除） |
| F-05 | 批量导入 | 支持 Excel/CSV 批量导入商品 |
| F-06 | 分类管理 | 商品分类的增删改查（树形结构） |

#### 模块二：入库管理
| 编号 | 功能 | 描述 |
|------|------|------|
| F-07 | 入库单列表 | 分页展示入库记录 |
| F-08 | 新增入库 | 选择商品、填写数量、单价、供应商、经办人、备注 |
| F-09 | 入库单详情 | 查看单条入库单的完整信息 |
| F-10 | 入库冲销 | 对错误入库进行红冲处理 |

#### 模块三：出库管理
| 编号 | 功能 | 描述 |
|------|------|------|
| F-11 | 出库单列表 | 分页展示出库记录 |
| F-12 | 新增出库 | 选择商品、填写数量、领用人/部门、经办人、备注 |
| F-13 | 出库单详情 | 查看单条出库单的完整信息 |
| F-14 | 出库冲销 | 对错误出库进行红冲处理 |
| F-15 | 库存校验 | 出库时自动校验库存是否充足 |

#### 模块四：库存查询
| 编号 | 功能 | 描述 |
|------|------|------|
| F-16 | 实时库存 | 展示当前所有商品库存量、库存金额 |
| F-17 | 多条件搜索 | 按商品名称、编码、分类模糊搜索 |
| F-18 | 库存预警 | 高亮显示低于安全库存的商品 |
| F-19 | 出入库明细 | 按商品/时间范围查看出入库流水 |

#### 模块五：库存盘点
| 编号 | 功能 | 描述 |
|------|------|------|
| F-20 | 创建盘点单 | 选择盘点范围（全部/按分类）生成盘点任务 |
| F-21 | 录入实盘数 | 对照系统库存填写实际盘点数量 |
| F-22 | 盘点差异 | 自动计算盈亏差异，生成差异报告 |
| F-23 | 盘点确认 | 确认后自动生成盘盈入库单/盘亏出库单调整库存 |

#### 模块六：打印表单
| 编号 | 功能 | 描述 |
|------|------|------|
| F-24 | 入库单打印 | 生成入库单 PDF / 浏览器打印 |
| F-25 | 出库单打印 | 生成出库单 PDF / 浏览器打印 |
| F-26 | 盘点报告打印 | 生成盘点差异报告 |
| F-27 | 库存报表打印 | 生成库存汇总报表 |

#### 模块七：系统管理
| 编号 | 功能 | 描述 |
|------|------|------|
| F-28 | 用户管理 | 系统用户的增删改查 |
| F-29 | 角色权限 | 基于角色的访问控制（管理员/仓管/只读） |
| F-30 | 操作日志 | 记录关键操作（入库/出库/盘点/用户变更） |

## 七、关键技术方案

### 7.1 库存扣减策略

采用**实时扣减 + 数据库事务**方案：

```
入库流程：
BEGIN TRANSACTION
  → 创建 stock_in 记录
  → 逐条创建 stock_in_item
  → 逐条更新 products.current_stock += quantity
  → 记录操作日志
COMMIT

出库流程：
BEGIN TRANSACTION
  → 校验每项库存是否充足（FOR UPDATE 行锁）
  → 创建 stock_out 记录
  → 逐条创建 stock_out_item
  → 逐条更新 products.current_stock -= quantity
  → 记录操作日志
COMMIT
```

### 7.2 冲销策略

- 入库冲销：生成一条**负数量**的出库记录（标记 `reversed_from`）
- 出库冲销：生成一条**负数量**的入库记录（标记 `reversed_from`）
- 原单标记 `status = 2`（已冲销）
- 确保库存平衡：`SUM(入库) - SUM(出库) = current_stock`

### 7.3 盘点差异处理

确认盘点后，系统自动：
1. `actual_qty > system_qty` → 生成盘盈入库单
2. `actual_qty < system_qty` → 生成盘亏出库单
3. 批量更新 `products.current_stock`

### 7.4 打印方案

- 前端使用 `print-js` 库 + 自定义 CSS `@media print` 样式
- 复杂报表（盘点报告）可由后端生成 PDF（`barryvdh/laravel-dompdf`）
- 打印模板使用独立组件，确保格式规范

### 7.5 安全性

| 措施 | 说明 |
|------|------|
| JWT Token | 有效期 2 小时，支持 Refresh |
| 密码加密 | bcrypt (cost=12) |
| SQL 注入防护 | Laravel Eloquent 参数绑定 |
| XSS 防护 | Vue 默认转义 + 输入过滤 |
| CORS | 配置白名单域名 |
| 操作日志 | 所有写操作记录到 `operation_logs` |
| 权限控制 | 后端中间件校验角色 |
